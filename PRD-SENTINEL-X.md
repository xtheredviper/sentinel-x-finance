# PRD: SENTINEL-X - Financial Defense System (v2.2)

## 1. CONCEITO E VIBE
O **SENTINEL-X** é um sistema de monitoramento de integridade financeira inspirado em terminais de segurança cibernética. Ele trata a saúde financeira como a integridade de um servidor crítico: gastos impulsivos são considerados ameaças ou intrusões, e economias são tratadas como upgrades de hardware.

> **PRINCÍPIO DE DESIGN UNIVERSAL:** A interface deve ser projetada para alta legibilidade e contraste, permitindo acessibilidade visual. O sistema deve suportar entradas de texto simples e comandos de voz, garantindo que usuários com diferentes níveis de letramento digital ou limitações motoras possam operar o firewall financeiro com eficiência.

## 2. O PROBLEMA
A fadiga de dados em aplicativos financeiros tradicionais resulta em abandono do controle. O usuário médio sente que preencher planilhas é um processo burocrático. O SENTINEL-X resolve isso transformando o registro de gastos em uma missão de defesa ativa e proteção de ativos.

## 3. PERSONA DO AGENTE: SENTINEL-X
- **IDENTIDADE:** Inteligência Artificial de monitoramento tático e proteção de ativos.
- **TOM DE VOZ:** Vigilante, direto e focado em métricas de sistema. Utiliza analogias de computação para descrever cenários financeiros.
- **AÇÕES DE RESPOSTA:** Identifica *buffer overflows* (gastos acima do limite), sugere *patches* de correção (cortes de gastos) e reporta o status de saúde do kernel financeiro.

## 4. FUNCIONALIDADES-CHAVE (PROTOCOLOS)
1. **DIRECT COMMAND INPUT:** Registro de transações via linguagem natural simulando comandos de terminal. Exemplo: `"sentinel, log: almoço 45cr"`.
2. **CONFIGURATION PROTOCOLS:** Capacidade de criar regras e objetos via chat. O usuário pode solicitar: `"Crie uma meta de reserva de emergência de 5000cr"` ou `"Defina um limite de 200cr para lazer"`.
3. **TERMINAL CONTROL (CLEAR):** Botão de ação rápida ou comando `"clear"` para limpar o histórico visual da conversa (**Purge Log**), mantendo a interface organizada e focada na operação atual.
4. **PERIMETER ALERT:** Notificações de intrusão quando o usuário se aproxima do limite crítico de uma categoria ou objetivo.
5. **UPGRADE PATH (METAS):** Objetivos financeiros visualizados como módulos de expansão de hardware. O progresso é exibido como carregamento de dados.
6. **KERNEL HEALTH CHECK:** Dashboard de status que resume a estabilidade financeira nos níveis: **OTIMIZADO**, **ALERTA** ou **CRÍTICO**.

## 5. ENTREGÁVEIS SOLICITADOS (UI/UX MINIMALISTA E ACESSÍVEL)
- **ESTRUTURA DE TELAS:** Terminal de Comando (Chat com opção de Clear), Monitor de Status (Dashboard de saúde) e Firewall de Objetivos (Metas).
- **SIMULAÇÃO DE INTERAÇÃO:**
**Usuário:** "sentinel, limpe o terminal."
**SENTINEL-X:** "COMANDO EXECUTADO. Log visual purgado. Sistema operacional em background mantido."

- **PLANO DE MVP:** Validar o engajamento através da estética de segurança de sistemas e testar a usabilidade da interface baseada em texto para criação de metas e controle de terminal.
