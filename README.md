# SENTINEL-X: Financial Defense System 🛡️

## 1. Project Description 📁
**SENTINEL-X** is a conceptual personal finance organization system developed under the **Vibe Coding** methodology. Unlike traditional apps based on spreadsheets and forms, SENTINEL-X utilizes a natural language conversational interface, inspired by Cyberpunk aesthetics and cybersecurity terminals.

The system treats the user's savings as the integrity maintenance of a critical server: excessive spending is logged as system threats (intrusions), while reaching financial goals is visualized as hardware upgrades and module expansions.

## 3. Creation Process (Vibe Coding) 🧠
The development of this project followed three main stages of AI interaction:

1. **Context Definition:** Establishing a high-tech visual identity, merging Cybersecurity concepts with personal financial management to reduce friction in data logging.
2. **Prompt Engineering (PRD):** Elaborating a detailed Product Requirements Document, instructing the AI to act as a systems architect and an interface designer focused on UX.
3. **Iterative Refinement:** Adjustments to the persona's behavior to ensure a technical and vigilant tone of voice, prioritizing **Universal Design** to ensure the interface is readable and operable by any user.

## 4. Main Features ⚙️
- **Direct Command Input:** Transaction logging via chat simulating console commands (e.g., `log: coffee 10cr`).
- **Configuration Protocols:** Creation of goals and budget limits directly through conversation (e.g., "sentinel, create an emergency fund goal").
- **Terminal Control:** Visual cleanup function (**Purge Log**) to maintain focus on the current system operation.
- **Perimeter Alert:** Tactical notification system for spending that compromises the financial kernel health.
- **Health Check:** Minimalist dashboard summarizing financial status in levels: **Optimized**, **Alert**, or **Critical**.

## 5. Reflection on the Process 📝
- **What worked well:** The AI successfully translated complex system security analogies into financial functions, creating a gamified and immersive experience.
- **Challenges:** Balancing the terminal aesthetics (scanlines and monospaced fonts) with Universal Design guidelines to maintain accessibility.
- **Learning:** Using **Vibe Coding** allowed focusing on architecture and product intent, leaving visual and functional execution to the creative partnership with AI.

## 6. Technologies and Tools 🛠️
